# 应用图标设计规格（ICON_DESIGN）

> 方向二「多音符汇入歌单」——在保留网易云「红底+白音符」品牌血缘的同时，强调本工具的核心卖点：**批量把歌曲/专辑加入歌单**。

## 一、设计概念

核心叙事：**多个音符从右上方汇成一道流，导入左下方的歌单卡片**——用流动方向和数量感传达「批量加歌」。

## 二、配色规格

| 用途 | 颜色 | Hex | 说明 |
| --- | --- | --- | --- |
| 主背景（圆角方/圆底） | 网易红 | `#C20C0C` | 网易云经典主红，品牌血缘 |
| 背景渐变辅色 | 深红 | `#E60026` → `#A50808` | 左上到右下轻微渐变，增加立体感 |
| 音符 / 主图形 | 纯白 | `#FFFFFF` | 高对比，保证小尺寸辨识 |
| 音符流轨迹线 | 半透明白 | `#FFFFFF` @ 30% | 表达「流动 / 批量」动势 |
| 歌单卡片底 | 浅白 | `#FFF5F5` | 与纯白音符区分层次 |
| 歌单卡片描边/列表线 | 浅红灰 | `#F2B8B8` | 弱化的列表行示意 |
| 「+」徽标底（可选） | 亮红 | `#FF3A3A` | 强调「添加」动作 |

配色逻辑：整体维持网易红+白的双色体系，仅在歌单卡片和轨迹线上做微弱明度变化，避免小尺寸下糊成一团。

## 三、构图比例（基于 1024×1024 画布）

```
┌─────────────────────────────┐  ← 1024 × 1024 安全画布
│   ┌───────────────────┐     │
│   │  ◯ ◯ ◯  音符流     │     │  ← 上方 40%：3 个音符
│   │     ↘ ↘ ↘         │     │     沿对角线汇入
│   │        ↘          │     │
│   │   ┌──────────┐    │     │
│   │   │ ▤▤▤▤▤▤  │    │     │  ← 下方：歌单卡片
│   │   │ ▤▤▤▤▤▤ +│    │     │     带列表线 + 加号角标
│   │   └──────────┘    │     │
│   └───────────────────┘     │
└─────────────────────────────┘
```

关键比例（参考 iOS/macOS 图标规范）：

- **画布**：1024 × 1024 px
- **圆角底**：圆角半径 `224px`（约 22%，macOS Big Sur 风格）；若做 Android/通用可用纯圆形。
- **安全内边距**：四周留白 `≥ 100px`，主元素控制在中心 `824 × 824` 区域内。
- **音符大小**：单个音符高约 `180px`，三音符递减（180 / 150 / 120）制造由远及近的纵深。
- **音符排布**：沿 45° 对角线分布，间距约 `120px`，从右上 (≈70%,25%) 流向左下 (≈40%,60%)。
- **歌单卡片**：宽 `420px` × 高 `300px`，居于下方偏左，圆角 `40px`，内含 3–4 条列表线（行高约 `50px`）。
- **加号角标**：直径 `160px` 的圆，叠在卡片右下角，「+」线宽 `28px`。

## 四、视觉细节建议

1. **动势优先**：音符流的对角线方向必须明确指向卡片，让人一眼看出「汇入」而非随机散落。
2. **层次区分**：纯白音符 vs 浅白卡片，靠明度差（100% vs 96%）区分，不要靠描边堆细节。
3. **小尺寸验证**：缩到 64×64、32×32 时，应仍能辨认「音符 + 卡片」，过密的轨迹线在 32px 下建议自动隐藏（多分辨率出图）。
4. **导出规格**：1024（源）、512、256、128、64、32、16 全套；macOS 用 `.icns`，Windows 用 `.ico`，Electron 应用图标建议保留 1024 PNG 源 + 平台打包格式。
